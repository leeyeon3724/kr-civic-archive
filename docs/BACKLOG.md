# 백로그

이 문서는 앞으로 진행할 작업만 관리합니다.

## 운영 원칙

- 완료 항목은 이 문서에 남기지 않습니다.
- 완료 이력은 기본적으로 `git log`로 관리합니다.
- 릴리스 노트가 필요한 변경은 `docs/CHANGELOG.md`에 기록합니다.
- 각 항목은 상태(`계획`, `진행 중`, `차단`)를 명시합니다.

## 현재 백로그

- [계획] [SEV-2] `app/observability.py`, `app/__init__.py` - `/metrics` 인증/인증 정책 결합 위험
  - 증상: 일반 API 보호 의존성을 그대로 사용해 운영 정책 변경(보호 토글) 시 모니터링 경로 가용성·보안 균형이 동시에 흔들림
  - 위험: 요구사항 분리 미비로 운영 측 모니터링 수집/차단 정책 전환 시 장애 대응 시간 증가
  - 검증: `/metrics` 접근 정책 전용 의존성 모델 및 테스트 추가

## 운영 메모

- 새 작업 추가 시 기존 항목과 중복 여부를 먼저 확인합니다.
- 작업 완료 시 본 문서에서 항목을 제거하고, 필요하면 `docs/CHANGELOG.md`에만 기록합니다.
